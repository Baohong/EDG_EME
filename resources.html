Frequently Asked Questions

  • Why am I having trouble installing EME?

    • The EME may be run as a standalone product or as an ArcGIS Extension. In order to install EME and run in either mode, Microsoft .NET Framework 3.5 or higher must be installed on your computer. Microsoft's .NET Framework 3.5 is freely available. As of 3/21/2014, the website for the download is here .

    You can determine if the .NET Framework is on your computer by going to Start -> Control Panel -> Add or Remove Programs.

    You should see Microsoft .NET Framework 3.5 listed. Multiple versions of Microsoft .Net Framework (e.g., 1,2,3) may exist on your computer simultaneously. If you plan to use the EME as an ArcGIS Extension, then you must also have the following:

    • ArcGIS 10 or higher must be installed on your computer.
    • The FGDC Patch for ArcGIS must be installed on your computer if you are running ArcGIS 10.0 and have not installed Service Pack 1. The FGDC Patch for ArcGIS is free and is available here .

  • What language is EME written in?

    • EME is written in VB.Net. For more information on the full EME technology stack, please see the about page.

  • Can different versions of .NET Framework coexist?

    • Yes.

  • Why can't I edit the coordinate system in EME?

    • The EME supports Geographic, State Plane, UTM, Web Mercator, and two versions of the Albers coordinate systems/projections. When a metadata record contains content that describes one of these supported systems, the EME allows the user to edit the information. If the coordinate system described in the metadata record is not one of these five, then the EME controls are locked and the user is not able to edit the coordinate system content. Because this element is fairly complex, if a metadata record contains coordinate system information that the EME does not recognize, it does not allow the user to edit that information. It is recommended that individuals use the EPA Synchronizer to update coordinate system information in their metadata record.

  • Why don't I see my database edits in EME?

    • The EME creates a user copy of the database in each user's user directory. This change was implemented in EME v3.1 so that individuals with managed desktop computers would be able to edit the database. More information about the EME database and where to find it on your computer is located in the EME help in the section titled 'Setting up the EME Database'.

  • Why do I see "Read Only" for FGDC CSDGM metadata in the metadata description tab?

    • As of ArcGIS 10, Service Pack 3, Esri made a number of changes to the native ArcCatalog metadata editing process. FGDC CSDGM Metadata is still editable through the EME. *Upgrading* metadata will convert your FGDC CSDGM elements to ArcGIS elements, but is not required to allow you to edit FGDC metadata in the EME.
